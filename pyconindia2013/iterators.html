Produce iterators that are memory-efficient

1 # Example - Fibonacci function
2 def fibonacci(n):
3     """ Fibonacci number using recursion """
4 
5     if n in (0, 1):        
6         return 1
7     else:
8         return fibonacci(n-1) + fibonacci(n-2)

Problems

  • Stack heavy and memory inefficient
  • Needs to compute all (n-1)th fibonacci numbers to compute the nth fibonacci number.

Fibonacci

1 def fibonacci(n):
2     """ Fibonacci generator function """
3 
4     x, y = 1, 2
5     for in range(n):
6         yield x            
7         z = x + y
8         x, y = y, z

Generator Syntax

  • Produces an iterator
  • Use yield instead of return to produce an output
  • Stack is frozen at yield and re-entered in next iteration

Pros

  • Memory efficient iterators
  • Lazy computation - Produces values at request
  • State keeping is taken care of automatically

Cons

  • Slightly cryptic code
  • Needs rewriting of functions sometimes

Filter data using selectors

itertools.compress filters elements from a sequence by applying a list of selectors which evaluates to True. Can be used to apply a list of filters against a list of elements.

 1 # Drop every alternate letter in a string of alphabets
 2 import string
 3 
 4 for i in itertools.compress(string.letters, itertools.cycle([1,0])):
 5     print i,
 6 
 7 a c e g i k m o q s u w y A C E G I K M O Q S U W Y
 8 
 9 # Drop students who failed to get marks above a cut-off
10 # in Python programming exam.
11 marks = {'Kiran': 93, 'Anand': 85, 'Soujanya': 92,
12          'Manohar': 98, 'Rakshith': 90, 'Nisha': 20}
13 # Make sure the list is sorted in both cases
14 selectors = [marks[student]>27 for student in sorted(marks)]
15 
16 list(itertools.compress(sorted(marks), selectors))

Filtering elements based on condition

itertools.ifilter returns an iterator that applies a condition to elements in an iterator and returns elements that match the condition.

1 # An infinite iterator for odd numbers
2 for number in itertools.ifilter(lambda x: x%2, itertools.count()):
3     print number
  • itertools.ifilter is similar to filter but returns an iterator instead of a list.
  • itertools.ifilterfalse performs the reverse operation, i.e returns an iterator for which the condition evaluates to False.

More magic with iterators

You often need to write code that returns till it finds a sentinel value. 

 1 flag, data = False, []
 2 # Keep producing random numbers till a specific value
 3 sentinel = 49
 4 
 5 while True:
 6    num = random.randrange(1, 100)
 7    if num == sentinel: 
 8       break
 9    data.append(num)
10 
11 # Idiomatic
12 import functools
13 
14 # Freeze function with all its arguments
15 func = functools.partial(random.randrange, 1,100)
16 
17 for num in iter(func, sentinel):
18     data.append(num)

A generator expression is similar to a list comprehension, except that the expression produces an iterable generator, instead of a list.

Instead of the square brackets of a list comprehension, a generator expression uses parens.

1 # Generator expression producing an infinite
2 # square iterator.
3 ((x*x for x in itertools.count())
4 
5 # Generator producing an infinite list of odd numbers
6 ((x for x in itertools.count() if x%2))

Rule of thumb - Generator expressions vs List comprehensions

Use list comprehensions where,

  • The actual list or sequence you produce out of the list comprehension matters.
  • Your list or sequence is small enough to fit into memory.

Use generator expressions where,

  • The final output is what that matters instead of the actual list or sequence. E.g: Functions like sum, all, any etc.
  • You want to create quick generators using simple conditions without having to define a function.
  • When dealing with huge sequences where saving memory is important.

Summary

  • Iterators are at the heart of Python.
  • Iterators can be seen in core data structures (lists, dictionaries, tuples, strings, sets) as well as in language features like list comprehensions, generators and generator expressions.
  • Use for loops for iteration as it "knows" how to work with iterators in Python.
  • List comprehensions provide a concise syntax for producing lists.
  • Generators produce memory-efficient, lazily computing iterators.
  • The itertools module provides a rich library of generator functions.
  • Generator expressions provide a concise syntax for writing generators.
